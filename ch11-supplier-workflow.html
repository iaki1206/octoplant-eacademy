Supplier Workflow – External Collaboration

When an external supplier needs to modify a project, Octoplant's Supplier Workflow provides a secure, traceable process for exporting the project, having the supplier make changes, and re-importing the results with full version tracking.

⏱ 4:46 min 🎯 Collaboration 📋 7 Steps
Tutorial Video – iOpen e-ACADEMY: Supplier Workflow – External Collaboration
1

Supplier Check-Out: Exporting the Project

The Supplier Check-Out function packages the project files for export to an external supplier who does not have direct access to the Octoplant server.

1
Select the component in the Project tree and click Check-Out in the Home tab.
2
In the Check-Out dialog, move the component to the Selected components area.
3
Activate the Supplier Check-Out (user-defined target directory) checkbox.
4
Activate Lock for other users — this is strongly recommended to prevent conflicting changes while the supplier is working.
5
Optionally activate Compressed to create a ZIP archive, and Use project tree directory structure when exporting multiple components.
6
Click the ... button to select the export target directory.
7
Click Check-Out and close. The exported package contains the project files plus three automatically generated files:
• SupplierChangeReason.txt — for the supplier to document change reasons
• SupplierComment.txt — for general comments
• SupplierDeliveryNote.vdog-sdn — do not delete this file!
🚫
Critical FileThe SupplierDeliveryNote.vdog-sdn file is required for the SmartImport process. If the supplier deletes it, the re-import cannot be completed. Instruct your supplier to keep all files intact.
2

SmartImport: Re-Importing Supplier Changes

When the supplier returns the modified project, use SmartImport to re-integrate the changes with full comparison and version tracking.

1
In the Home tab, click SmartImport for supplier projects.
2
In the SmartImport dialog, click Add and select the directory containing the supplier's returned files (must contain the .vdog-sdn file).
3
Activate the Import checkbox for the projects you want to import.
4
Select the comparison type to review the supplier's changes:
• Supplier data ↔ Original version (recommended): Shows what the supplier changed
• Supplier data ↔ Working directory: Compares with your current local files
5
Select the import action: Create versions and Check-In (recommended) to immediately create a new version and upload it to the server.
6
Click Import. Octoplant creates a new version documenting the supplier's changes and uploads it to the server.
