Asset Inventory Service

The Asset Inventory Service is a security-focused feature that automatically identifies device information and vulnerability data for all managed assets. This chapter shows how to access device details, read vulnerability reports, and interpret CVE risk scores.

⏱ 3:22 min 🎯 Security 📋 4 Steps
Tutorial Video – iOpen e-ACADEMY: Asset Inventory Service
1

Triggering Asset Identification via Check-In

The Asset Inventory Service is automatically populated when components are checked into the server. Octoplant reads the CPU configuration from the project data.

1
In the UserClient, select a component (e.g., a Simatic S7 component) and perform a Check-In with project data.
2
Octoplant automatically reads the CPU configuration from the project data and registers it in the Asset Inventory Service.
2

Accessing the Asset Inventory Dashboard

The Asset Inventory Service is accessible from the octoplant Hub.

1
Log into the octoplant Hub (see Chapter 8).
2
Navigate to the Asset Inventory Service dashboard. A list of all managed assets appears, showing: Name, IP address, Risk level (Low / Medium / High), Device type, Vendor.
3
A color-coded donut chart provides an overview of the vulnerability distribution across all assets.
3

Viewing Device Information

Click on any asset to see its full technical profile.

1
Click on an asset in the list to open its Device Information page.
2
Review the comprehensive device details: network configuration, hardware information (including rack slots and module assignments), and installed firmware/software versions.
4

Reading Vulnerability Reports

The vulnerability report provides a detailed security assessment for each device.

1
From the Asset Inventory dashboard, click on the risk level (e.g., 'Medium') next to a specific asset.
2
The detailed risk report opens, showing: overall Risk Score, a Radar Chart visualizing risk across categories (Vulnerability, Threat, Criticality), and a breakdown of each category.
3
Scroll down to the Insights section to see a list of Full Match CVEs (Common Vulnerabilities and Exposures). Each CVE entry shows its score, title, and date identified.
ℹ️
CVE InformationCVE data is sourced from the National Vulnerability Database (NVD). Octoplant matches your device's firmware version against known CVEs to identify potential security risks.
4
Click on any CVE entry to see more details and recommended mitigations.
