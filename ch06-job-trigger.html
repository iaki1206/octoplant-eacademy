Job Trigger and Verification

Once jobs are configured, you need to know how to manually trigger them, monitor their execution, and interpret the results. This chapter covers manual job execution, reading job results, comparing server versions with backups, and understanding email reports.

⏱ 3:47 min 🎯 Operations 📋 5 Steps
Tutorial Video – iOpen e-ACADEMY: Job Trigger and Verification
1

Manually Triggering a Job

While jobs run automatically on schedule, you can also trigger them manually at any time — for example, after making changes to a device.

1
In the UserClient, select the component associated with the job in the Project tree.
2
Navigate to the Jobs tab in the menu bar.
3
Click the Execute jobs once button. The job starts immediately.
4
Click the Show progress information button (which becomes active during execution) to monitor the real-time execution status.
2

Reading Job Results

After a job completes, the results are available in both the AdminClient and UserClient.

1
In the Jobs tab of the UserClient, select the component and review the job history table.
2
Look at the Server version ↔ Backup column:
• Equal: The backup matches the server version — no changes detected
• Different, with warning: The backup differs from the server version — a change was detected on the device
3
Look at the Prev. backup ↔ Backup column to see if the device program changed since the last backup.
⚠️
Not Present WarningIf a previous backup was unavailable for comparison (e.g., first run), you will see a 'Not present, with warning' status. This is normal for the first execution.
3

Comparing Server Version with Backup

When a 'Different' status appears, you should investigate the differences before deciding how to proceed.

1
In the job results, right-click on the result row that shows a 'Different' status.
2
Select Compare server version with backup from the context menu. A detailed comparison window opens.
3
Review the side-by-side Difference tree to understand exactly what changed on the device. Based on this analysis, decide whether the device change was authorized or unexpected.
4

Understanding Email Reports

If email notifications are configured, Octoplant sends automated reports after job execution.

1
Open the notification email in your email client. The email contains a summary of all job results.
2
Download the attached CSV file for a detailed breakdown of all differences detected across all monitored components.
3
Use the CSV data to create reports, track trends, or integrate with other systems.
