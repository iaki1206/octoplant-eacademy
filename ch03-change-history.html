Explanation of Change History

The Change History is Octoplant's complete audit trail — a chronological record of every version ever created for a component. This chapter explains how to read the change history, navigate between versions, view modified elements, and use the history for compliance and troubleshooting.

⏱ 3:54 min 🎯 Core 📋 4 Steps
Tutorial Video – iOpen e-ACADEMY: Explanation of Change History
1

Accessing the Change History

The Change History is accessible from the Component details panel whenever a component is selected in the Project tree.

1
In the Project tree, select any component that has been checked in at least once.
2
In the central panel, click the Change history tab. A list of all versions appears, showing version number, date, user, and comment.
3
To customize which columns are displayed, right-click on any column header and check/uncheck the columns you want to see (e.g., Change reason, Duration, File size).
2

Checking Out Specific Versions

You can check out any specific historical version of a component, not just the latest one. This is useful for comparing or restoring older states.

1
In the Home tab, click Check-Out. The Check-Out dialog opens.
2
In the dialog, expand the component to see all available versions. Select the specific version(s) you want to retrieve locally.
3
Click Check-Out and close. The selected versions are copied to your local archive. The latest version is always placed in the working directory.
ℹ️
Working DirectoryOnly the most recent checked-out version is placed in the working directory (the folder you can edit). Older versions are stored in the local archive for reference and comparison only.
3

Viewing Modified Elements

For each version in the Change History, you can see exactly which elements were modified — down to the individual program block or data point.

1
In the Change history tab, click on a specific version to select it.
2
The Modified elements table below the version list updates to show all elements that changed in that version, including the type of change (added, modified, deleted).
3
Click on any modified element to see more detail about the specific change.
4

Viewing Component Details and Master Data

The Details tab provides additional metadata about the component that can be used for searching and documentation.

1
With a component selected, click the Details tab in the central panel.
2
Review the Component ID — a unique identifier assigned by Octoplant that can be used to reference this component in reports and exports.
3
View or edit the Master data fields. These are custom metadata fields (e.g., IP address, location, responsible engineer) that can be used to search for components across the entire project tree.
