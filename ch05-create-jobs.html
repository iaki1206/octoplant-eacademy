Create Jobs – Automated Backup Configuration

Jobs are the heart of Octoplant's automated backup system. A job defines how, when, and from which device Octoplant automatically reads and archives program data. This chapter walks through creating a component, then configuring a full job with schedule, save policy, compare policy, and email notifications.

⏱ 5:24 min 🎯 Admin 📋 8 Steps
Tutorial Video – iOpen e-ACADEMY: Create Jobs – Automated Backup Configuration
1

Prerequisite: Create and Check-In a Component

Before creating a job, the target component must exist in Octoplant and be checked into the server.

1
Create a new component in the UserClient (see Chapter 1 for detailed instructions). For example, create a Simatic S7 component named S7_300_Line1.
2
For Siemens S7 components, you must have project data checked in before the job can read from the device. Drag the project files into the component and click Create base version and Check-In.
ℹ️
NoteFor some component types (e.g., Image Service), you can create a job without pre-existing project data. For Siemens S7 and similar types, project data must be present.
2

Creating a Job in the AdminClient

Jobs are created and configured in the AdminClient, not the UserClient.

1
Open the Octoplant AdminClient and click on Jobs in the left navigation panel.
2
Locate the component you want to create a job for in the Project tree on the right side.
3
Drag the component from the Project tree and drop it into the Jobs view on the left. A new job is automatically created for that component.
💡
AlternativeYou can also click the Create button in the Jobs view and then manually select the component. The drag-and-drop method is faster.
3

Configuring the Job

With the job created, configure its settings to define how it behaves.

1
Enter a unique, descriptive Job name (e.g., Daily_Backup_S7_300_Line1).
2
Set the Upload type to define how data is read from the device (e.g., via PROFIBUS, Ethernet, or a specific protocol).
3
Enter the IP address of the physical device (e.g., 192.168.1.100).
4

Configuring the Schedule

The schedule defines when the job runs automatically.

1
Click on the Schedule field to open the schedule configuration. The default is daily at 1:00 AM.
2
Modify the schedule as needed. For example, set it to run weekly on all days at 11:00 PM for a nightly backup.
3
Click OK to confirm the schedule.
5

Configuring Save and Compare Policies

These policies control what is saved and what is compared after each job execution.

1
In the Save policy section, select Always save backup to ensure a backup file is created in the Octoplant backup folder every time the job runs, regardless of whether changes are detected.
2
In the Compare policy section, select the comparison types to perform after each backup:
• Server version ↔ Backup: Compares the current server version with the newly created backup
• Previous backup ↔ Backup: Compares the last two backups to detect changes on the device
6

Configuring Email Notifications

Octoplant can automatically send email reports after job execution.

1
Click on Global job settings to access the notification configuration.
2
Select the notification mode:
• Job specific: Sends an email about this specific job
• Daily: Sends a daily summary email about all configured jobs
3
In the Users to notify field, select the Octoplant users who should receive notifications. Their email addresses must be configured in User Management.
⚠️
PrerequisiteEmail addresses must be configured for each user in the User Management module before they can receive job notifications.
4
You can also add external email addresses for recipients who do not have Octoplant accounts.
5
Click Save to finalize the job configuration. Octoplant will now execute the job automatically according to the configured schedule.
