Active Directory Integration

Octoplant can synchronize with your organization's Active Directory (AD) to automatically import users and allow them to log in with their domain credentials. This chapter covers the complete AD configuration process.

⏱ 5:57 min 🎯 Enterprise 📋 7 Steps
Tutorial Video – iOpen e-ACADEMY: Active Directory Integration
1

Configuring the AD Connection

All AD configuration is done in the AdminClient's User Management module.

1
Open the Octoplant AdminClient and navigate to User Management → Synchronization.
2
Enter the AD server IP address, port number (typically 389 for LDAP or 636 for LDAPS), and the credentials of an AD service account with read access.
3
Use the built-in AD Explorer tool to browse your Active Directory structure and identify the correct root paths for the user groups and admin groups you want to synchronize.
4
Copy the root paths into the corresponding fields in the Synchronization settings.
5
Click Test connection to verify that Octoplant can communicate with the AD server. A success message confirms the connection.
⚠️
Connection TestAlways perform the connection test before attempting a user import. A failed test indicates incorrect settings (wrong IP, port, or credentials) that must be corrected first.
2

Importing Users from Active Directory

With the connection verified, you can import users from AD into Octoplant.

1
Click Manual import to immediately import all users from the configured AD groups into Octoplant.
2
After import, review the newly created user accounts. You can modify their properties (e.g., password settings, group assignments) as needed.
3
Configure Automatic daily import to keep Octoplant synchronized with AD automatically. New AD users will be imported, and removed users will be deactivated.
3

Setting Authorization Policies

Configure how imported AD users authenticate in Octoplant.

1
In the user properties, set the authorization policy to allow login via domain password or operating system credentials.
2
With this setting, users can log into Octoplant using the same password they use for Windows — no separate Octoplant password required.
