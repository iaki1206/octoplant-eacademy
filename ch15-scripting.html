Scripting – Automation and Custom Actions

Octoplant's scripting feature allows you to automate custom actions — such as running validation checks, sending notifications, or executing external tools — triggered by events like opening a component for editing or clicking a custom button.

⏱ 4:37 min 🎯 Advanced 📋 6 Steps
Tutorial Video – iOpen e-ACADEMY: Scripting – Automation and Custom Actions
1

Preparing Script Files

A complete Octoplant script package consists of three files that must be prepared before configuration.

1
Create a .script file containing the script code (the logic to execute).
2
Obtain or compile the .exe application file that executes the script. Download the scripting package from my.auvesy-mdt.com.
3
Create a matching .ini file that configures which component types the script applies to.
4
Optionally, include an image file if you want to create a custom button with an icon.
2

Configuring Scripts in the AdminClient

Scripts are registered and configured in the AdminClient's Script Management module.

1
Open the AdminClient and navigate to Script management.
2
Copy the prepared script files to the appropriate directory:
• Client-side scripts: Executed on the user's PC
• Server-side scripts: Executed on the Octoplant server
3
In Scripting settings, enable scripting functionality globally.
3

Configuring Script Triggers

Scripts can be triggered by specific events in the Octoplant workflow.

1
In the script configuration, set the trigger event. For example, the Before Edit event triggers the script whenever a user opens a component for editing.
2
Configure the trigger to apply to specific component types using the .ini file.
4

Creating a Custom Button

You can add a custom button to the Extras tab in the UserClient to allow users to run scripts on demand.

1
Include the image file for the button icon in the script package.
2
Configure the button in the script settings. The button will appear in the Extras tab of the UserClient.
3
Users can click the button to manually trigger the script on the selected component.
💡
Online HelpFor more scripting examples and advanced usage, navigate to Help → Online help in the UserClient or AdminClient. The documentation includes sample scripts for common automation scenarios.
